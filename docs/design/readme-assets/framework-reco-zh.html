Why prompt-only breaks

快起步, 难复盘。

大多数 AI 编码体验在前 10 分钟很快, 但在第 10 天开始崩: 上下文丢失, 并行混乱, 需求漂移, 无法审计。

会话结束: 决策与上下文只留在聊天记录里。
并行开发: 缺少 team lead 路由与责任归属。
质量门禁: 测试与完成标准容易被跳过, 很难追溯。
Architecture

会话协同 + 文件化状态 + Git 可追溯。

Agent Teams 负责实时编排. 项目状态沉淀在文件系统与 Git 中, 让工作可以暂停, 恢复, 复审, 并持续演进。

Session layer: team lead 协调并行 lanes, 明确 owner 与交付节奏。
Project layer: tasks.json, progress.txt, docs/ 作为可恢复记忆。
Governance: .claude/rules/ 定义流程, blocker 协议, 完成清单与质量门槛。
Benefits

可恢复, 可并行, 可审计。

把即兴提示变成可重复的交付操作系统: 专业分工, 可追溯变更, 与可恢复执行历史。

Resume blockers 与下一步写进文件, 随时接力
Parallelize team lead 路由角色, 并行不打架
Audit docs + state + Git 让交付可复盘
